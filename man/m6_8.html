6.8. Генерація звіту форми 46.

Для генерації звіту за формою 46 необхідні файли бази даних для складів підрозділів. Причина цього полягає в тому, що окрема база даних не має інформації про рух товару того підрозділу, на який була видача товару. Для конкретної бази рух товару закінчується з накладною на видаток. Форма 46 вимагає знання руху у підрозділах. Тому для коректної побудови звіту потрібні або залучати бази від кожного підрозділу, або ці бази необхідно створити та вести централізовано. Для формування звіту програма надає широкий арсенал автоматизації: 1)видаткові накладні на підрозділ з БД бригади можуть бути експортовані як прибуткові накладні для складу підрозділу. 2)видаткові накладні підрозділа (витрата) можуть бути імпортовані з накладних та карт-звітів. 3)витрата підрозділу по днях може бути сформована по меню розкладці та кількості людей на котловому забезпеченні на кожен день. Зазначена автоматизація дозволяє вести бази даних для підрозділів із мінімальними витратами часу. Форму для генерації звіту показано на мал. 6.12.

Мал. 6.12. Форма для створення звіту за формою 46

Таблиця у зазначеній формі призначена для набору файлів баз даних для всіх підрозділів, які мають бути у звіті. Ця таблиця має два поля - одне для вставки повного шляху до конкретної бази даних, друге це список, що випадає, з якого слід вибрати підрозділ, що відповідає вставленій базі даних. Така система вибору необхідна тому що сам собою файл бази даних немає точної інформації для якого підрозділу він ведеться. Після введення всіх необхідних пар «файл бази-ім'я підрозділу» та вказівки періоду часу можна згенерувати звіт за формою 46, де кожна сторінка буде звітом про рух для однієї позиції з каталогу. У звіті будуть присутні лише ті позиції, якими був рух. Після генераціі звіту ви можете швидко перейти до сторінки с бажаною позицією використовуючи випадаючий список товарів.
