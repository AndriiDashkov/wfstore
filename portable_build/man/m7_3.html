7.3.Вкладка "Списання".

Ця вкладка містить функціонал для модифікаціі меню-розкладок які вдповідають видачі за тиждень. Це може бути корисно, коли треба провести списання за меню-розкладкою, але кількість людей було змінено, тому розкладка на видачу не відповідає сітуаціі: потрібна нова меню-розкладка, яка буде "вірівнювати" кількість видатку по неї з реальним видатком по складу з точки зору кінцевих залишків за тиждень. Цей функціонал не змінює інформацію в базі даних, тільки використовує інформацію про видаток та прибуток за тиждень для генераціі зміненої розкладки. Процес формування нової розкладки виглядає таким чином:
1) Спершу треба завантажити (імпортувати) залишки на початок тижня із зовнішнього Excel файлу або з поточного складу. Ці залишки можуть бути із зовнішньої программи, наприклад з ЗСУ Прод, чи можно використати залишки на кінець попереднього тижня з цієї программи, це не має значення. Ідея полягає в тому, щоб взяти залишки на початок тижня. Якщо ці залишки з поточної програми, то дві перші колонки в таблиці порівняння будуть однакові. Якщо ці залишки з програми ЗСУ прод, то можна сформувати звіт руху для кожного договору в ЗСУ прод за попередній тиждень, експортувати в Excel, а потом об'єднати ці звіти за допомогою генератора "Об'єднання договорів ЗСУ прод" (див. вкладку "Звіти", тип звіту "Генератори"). Результат об'єднання в формі файла Excel ідеально підходить для завантаження залишків на початок тижня. Завантажені залишки є початковою базою для формування нової меню-розкладки.
2) Після цього треба завантажити в таблицю "Вибір файлів меню-розкладок" одну чи кілька розкладок, по яким їде списання (був видаток); Розкладок, по которим буде списання, може буди декілька (наприклад для різних підрозділів). Тому в таблицю "Вибір файлів меню-розкладок" треба завантажити усі бажані раскладки, але розкладка для модифікаціі повинна бути останьою у списку. Усі розкладки повинні містить відповідну кількість людей.
3) Потiм вказати період та завантажити видаток та прибуток зі поточного складу (кнопка "Завантажити видаток"). Після цих операцій таблиця порівняння буде показувати наступні дані:
- колонка імпортованих залишків
- колонка залишків на поточному складі (якщо було викристано завантаження залишків з поточного складу, дані в цієї колонці будуть однаковими з попередньою.)
- колонки "Надійшло", "Вибуло", "Залишок" - ці дані беруться з поточного складу. Зверніть увагу якщо ви виконуєте порівняння з начальними залишками з іншої програми (наприклад ЗСУ Прод) то неявно мається на увазі що прибуток за вказанний тиждень однаковий з поточним складом.
- Залишок по розкладці - ця колонка це розрахований сумарний видаток згідно меню-розкладкам, вказаним в таблиці.
- Різниця - ця колонка містить різницю між залишком згідно розкладки та залишком на поточному складі.
- Можлива коррекція - дані у цієї колонці заявляються після генраціі нової розкладки, флк ні для кожного випадку ця коррекція можлива.
- Коррекція для виконання - в цієї колонці ви можете змінити коррекцію до бажаної.
4)Після цього можно проаналізувати різницю та створити нову розкладку автоматично (кнопка "Генерувати розкладку"). Усі операціі відображаються у відповідних колонках таблиці. Якщо якась корекція небажана, її можно скоригувати у відповідній колонці "Коррекція для виконання". Зверніть увагу, якщо було завантажено кілька розкладок в таблицю "Вибір файлів меню-розкладок", то базою для модифікаціі та генераціі зміненої розладки будет тільки остання розкладка в таблиці. Остання кнопка праворуч дозволяє зберегти залішки для роботи з наступним тижнем. Також зверніть увагу, що алгорітм блокує коррекцію такої позиціі як вода (код 4012), яйця ( код по каталогу 2056, 2057), а також тих позицій, коррекція для яких є дуже велика. Це потрібно робити безпосередньо в файлі меню-розкладки, якщо це необхідно.
5)Після генераціі нової меню-розкладки ви можете зберегти її у вигляді Excel файлу застосуванням відповідної кнопки ("Зберегти нову розкладку"), збережена розкладка буде мати дані з урахуванням змін в колонці "Коррекція для виконання". Майте на увазі, що збережений файл нової розкладки может мати невірні макроси (з тієї причини що ісходний файл був закритий паролем). Щоб отримати повноціний файл меню-розкладки, створіть пустий файл меню-розкладки вашими штатними засобами (ЗСУ Прод та т.і.) та зробіть в ньому завантаження із збереженого файлу корегованої розкладки.
